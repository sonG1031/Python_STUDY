[시퀀스 자료형]
▶︎ 리스트(list), 튜플(tuple), range, 문자열(str)과 같이 값이 연속적으로 이어진 자료형을 시퀀스 자료형(sequence types).
▶︎ 시퀀스 자료형으로 만든 객체를 시퀀스 객체.

[시퀀스 자료형의 공통 기능]
▶︎ 시퀀스 자료형은 공통된 동작과 기능을 제공
▶︎ 따라서 리스트, 튜플, range, 문자열 등의 시퀀스 자료형은 같은 문법을 사용

[딕셔너리]
▶︎ 연관된 값을 묶어서 저장하는 자료형
▶︎ {}안에 key : value 형식으로 저장하며 각 키와 값은 ,로 구분.

[컨테이너란 무엇인가요?]
▶︎ 파이썬에서는 데이터를 담는 자료형인 리스트, 튜플, 딕셔너리, 세트를 컨테이너(container)라 부름.

[딕셔너리는 키의 순서가 보장되나요?]
▶︎ 파이썬 3.5 이하에서는 키의 순서가 정해져 있지 않음.
▶︎ BUT, 파이썬 3.6부터는 딕셔너리를 생성했을 때와 키를 추가했을 때의 순서를 따르므로 순서가 보장됨.

[의사 코드] (doctor code 아님ㅋㅋ)
▶︎ 의사 코드(pseudo code)는 실제 프로그래밍 언어가 아닌 사람의 언어로 프로그래밍 언어를 표현한 것.



[if 조건문]
if 조건식:
    코드 # 실행할 코드는 반드시 들여쓰기
• 실행흐름
  ▶︎ if의 조건식이 True 이면 코드 실행,
    if의 조건식이 False 이면 Skip!

[pass?]
▶︎ pass라는 키워드는 아무 일도 하지 않고 그냥 넘어간다는 뜻.
▶︎ 본문에 아무 코드도 넣지 않으면 에러가 발하므로 형태 유지를 위해 사용.
